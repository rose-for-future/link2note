# 决策记录：语音转文字后端选型

> 日期：2026-06-28　状态：已采纳　影响：`scripts/transcribe.py`、`scripts/config.py`、`scripts/fetchers/podcast.py`

## 背景
信息处理 skill 需要把视频/音频转成文字。最初设计选了 **SenseVoice（阿里 FunASR）** 作为默认（中文质量最佳）。真实落地时连续踩坑，三次调整后定型。

## 踩坑过程（事实记录）
1. **SenseVoice 装不上**：`funasr` 依赖链里的 `llvmlite` / `cryptography` 需要从源码编译，在用户机器上（先是系统 Python 3.14，后是 miniconda x86_64 Python 3.12）反复失败。结论：funasr 全家桶环境太挑，不适合做"别人下载就能用"的默认。
2. **改 faster-whisper（CPU）**：装得上、跑得通，small 模型中文质量可接受。但**纯 CPU 转写极慢**——2 小时播客估计要 20–40 分钟，且首版 pip 默认想从源码编译 `av`，需 `--only-binary=:all:` 才走预编译 wheel。
3. **发现真正瓶颈：没用 GPU**。用户是 Apple Silicon，CPU 硬转浪费了 Metal GPU / 神经引擎。调研确认 **MLX（苹果框架）** 在 Apple Silicon 上跑 Whisper 比 CPU 快数倍。

## 决策
- **默认后端 = `auto`**：运行时检测 `Darwin + arm64 且装了 mlx_whisper` → 用 **mlx-whisper**；否则回退 **faster-whisper**。既给苹果用户最快体验，又保持跨平台可移植。
- **mlx-whisper 默认模型 = `mlx-community/whisper-large-v3-turbo`**：比 large-v3 快约 5x，中文准确度远高于 small，~6GB 运行内存，跑 Apple GPU/Neural Engine（Metal）。
- **faster-whisper 默认模型 = `small`**（非苹果平台兜底，可在 config 调 medium / large-v3）。
- **sensevoice 保留为可选**（中文质量参考），但**不作默认**，因安装门槛高。
- 安装纪律：MLX/faster-whisper 必须用 `pip install --only-binary=:all:`，避免源码编译失败；MLX 需 **arm64 原生** Python（x86_64/Rosetta 不行）。
- 顺带修复：播客音频 URL 的 `&amp;` 实体未转义 + curl 断流，已加 `html.unescape` 和 `curl --retry 3 --retry-all-errors`。

## 取舍理由
| 维度 | SenseVoice | faster-whisper(CPU) | **mlx-whisper(GPU)** |
|------|-----------|---------------------|----------------------|
| 安装稳定性 | 差(编译失败) | 好 | 好(arm64 wheel) |
| 速度 | 慢(CPU) | 慢(CPU) | **快(Metal GPU)** |
| 中文质量 | 最好 | small 一般 | turbo 很好 |
| 可移植 | 差 | 好 | 仅 Apple Silicon |

→ `auto` 同时拿到"苹果快"和"跨平台稳"。

## 待办 / 后续

- 后续若要更高中文准确度，可评估 mlx 上的 SenseVoice 移植或 large-v3 全量模型。
